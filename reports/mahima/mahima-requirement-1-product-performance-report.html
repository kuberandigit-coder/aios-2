Data Sources & Calculation Rules
Source: PostgreSQL staging_ai.cppc_workbook_product_performance_v1, filtered to campaign IDs 20763699505, 23684789991, 23053104908, 23431543574, 23926509987 (Mahima's ledsone.de campaigns, confirmed via staging_ai.cppc_campaign_truth_registry_v1, owner = "Mahi"). Read-only SQL only — no data modified.
CTR = Clicks ÷ Impressions · Avg CPC = Cost ÷ Clicks · Conv. Rate = Conversions ÷ Clicks · ROAS = Conv. Value ÷ Cost. All computed directly from the source row's spend/clicks/impressions/conversions/revenue fields.
Suggested Action: taken as-is from the source table's mahima_action column (pre-computed elsewhere; rule not yet reverse-engineered or confirmed with Mahima).
Known Limitations
1. Data is not live “last 30 days from today.” The freshest available data across every Google Ads/product table checked in PostgreSQL is dated 2026-06-11 (~4 weeks old). No Google Ads or Google Merchant Center API connector is available in this environment to pull live data; connecting one would require a Google-approved Developer Token and OAuth setup (see handover for details).
2. Product Price and Product Cost are Data Missing for every row. No table anywhere in the database has a working join from this dataset's product IDs to Shopify/Merchant Center pricing, and no product-cost/COGS source has real populated values (checked development.sku_cogs — empty; staging_ai.cppc_cogs_truth_model_v1 — all cost fields NULL). Gross Profit and Profit After Ads therefore cannot be calculated and are omitted from this version.
3. Feed Status and Missing Attribute are Data Missing. The correct source table (raw_data.gmc_product_diagnostics_daily) is completely empty (0 rows) — no Feed Status/Missing Attribute data exists anywhere in the database.
4. Last 7 Days ROAS / Last 30 Days ROAS are not shown separately. The source table stores one aggregate window per row with no day-level history to compute a rolling comparison from.
5. Product title is not available — shown as internal SKU (where matched, 94 of 680 rows) or the raw Google product/item ID otherwise.
6. Campaign coverage: only 4 of Mahima's 5 active ledsone.de campaigns have product-level rows in the source table; “Shopping DE | Mahi | klarna | TOP-MAHI | Verkaufsprodukt | tROAS | 11/06” (campaign 23926509987) has campaign-level totals but no product-level breakdown available.
7. Suggested Action rule not independently verified — see note above the table.
