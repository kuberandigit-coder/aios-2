Data source: two sources, both server-side only — PostgreSQL (read-only: google_ads.product_performance, google_ads.campaigns, google_ads.ad_group_products, listings.shopify_listings for SKU/URL/Image/Price) and the Shopify Admin GraphQL API (read-only, live Current Stock) · fetched via a server-side serverless function (/api/jefri/product-status) using credentials stored only as Vercel server-side environment variables — never present in this HTML or any client-side script.
ROAS: (Conv. Value ÷ Cost) × 100, calculated from aggregated 90-day totals per product (not averaged daily). Shown as “—” when Cost and Conv. Value are both zero.
Status: Merchant Center eligibility (Eligible / Disapproved / Limited) where available from Shopping/Search ad-group data. Performance Max campaigns do not expose per-product eligibility in this system, so those products show “Unknown” — this is a documented data gap, not a calculated guess.
Current Stock: fetched LIVE from the Shopify Admin GraphQL API on every load/refresh (not a database snapshot) — variant-level available inventory, summed across locations, for the advertised item (falls back to a representative child variant when the item ID is a parent-level product). Shows “—” when the item isn't inventory-tracked or has no resolvable Shopify listing.
Tag priority: Zombie is evaluated first (no impressions, no clicks), then Hero, then Villain, then Sidekick. Products matching none of the rules show ⚪ Unclassified rather than being forced into another category.
Currently-active products only: a product only appears here if it had performance data in the last 7 days — not just anytime in the 90-day window. Products removed from a campaign stop generating performance rows entirely, so recent silence reliably means "no longer in this campaign", matching what Google Ads' own live Products report shows. The displayed metrics (Impressions, Clicks, Conv. Value, Cost) still cover the full 90-day window for whichever products remain.
