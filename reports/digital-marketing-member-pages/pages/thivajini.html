Data Sources & Method

Google Ads Side (PostgreSQL)

  • Table: public.ppc_etl_performance_data
  • Filter: source=3, sub_source_id=233, record_type='campaign'
  • FR Verified: campaign names "Pmax FR | Thivajini…", EUR, LSFR prefix
  • Aggregation: DATE_TRUNC('week', date) — Monday-start ISO weeks
  • Coverage: 2026-04-06 → 2026-07-07 (14 weeks)

Shopify UTM Side (Shopify Admin API)

  • Source: Shopify Admin GraphQL API · ledsone.fr · 200 orders
  • Field: customerJourney.{firstVisit, lastVisit, moments}.utmParameters
  • Attribution: Any-touchpoint — utm_source="google_ads" & utm_medium="cpc"
  • Campaign: Last google_ads UTM touchpoint wins
  • Range: LSFR1303 – LSFR1488 (Apr 6 – Jul 8 2026) · EUR only

Campaign → UTM Mapping

  • 23103582865 ↔ tr-pmax-topsell: "Topsell" in both — high confidence
  • 23405519670 ↔ pmax_bestselling_tr: "Best Sellers / bestselling" — high confidence
  • 23533025729 ↔ pmax_allproduct: Remaining campaign — inferred, not confirmed via join
  • ⚠ No direct join table between campaign_id and UTM name in PostgreSQL

Known Limitations

  • Conversion window: Google Ads uses 30-day click window; Shopify uses session UTM — weeks will not align exactly
  • Multi-touch: Revenue assigned to last google_ads touchpoint only
  • Conversion counting: Google Ads may record >1 conversion per order
  • Thresholds: 0.95x–1.05x / 0.80x–1.20x — proposed, not confirmed

Decision Rules

  • Hero ROAS≥400% AND Clicks≥15 → Scale aggressively
  • Green ROAS≥400% AND Clicks<15 → Scale
  • Amber ROAS 300-399% → Maintain
  • Orange ROAS 250-299% → Optimise
  • Hi-Cut Orders>0 AND ROAS<250% → Review / Exclude
  • Bleeding Clicks≥5, CVR<1%, Orders=0 → Exclude
  • Low Eng. Impressions>0, Clicks=0 → Improve title / image
  • Zombie Impressions=0 → Check Merchant Center
  • Monitor Everything else → Monitor

KPI Formulae

  • CTR = (Clicks / Impressions) × 100
  • CVR = (Orders / Clicks) × 100
  • ROAS = (Sales / Spend) × 100
  • Spend% = (Spend / COALESCE(sale_price, price)) × 100
  • Spend Alert triggered when Spend% > 25%
  • NULLIF guards on all zero denominators

Business Rules

  • STOP Stock=0 AND 30d Spend>0 → Pause immediately
  • ACT SOON Stock≤5 AND Spend>0 → Cut budget / restock
  • MONITOR Stock≤5 AND Spend=0 → Watch
  • OK Stock>5 → No action needed
  • Low stock threshold: 5 units (default)
  • Wasted Spend: Only counted when Stock=0 AND Spend>0

Match Methods

  • Product ID match (144): ppc_etl.sku = gmp.product_id directly
  • Record ID match (102): ppc_etl.record_id ≈ gmp.product_id (case-insensitive)
  • Missing mapping (78): No gmp record found — feed data unavailable
  • Stock: listing_data.quantity via ref_id=variant_id bridge
  • Stock unknown (-1): No listing_data record; feed availability used as proxy
