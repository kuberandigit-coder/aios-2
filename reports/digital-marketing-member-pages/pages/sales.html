Data source: Shopify Admin GraphQL API (read-only), store ledsone.co.uk (ledsone.myshopify.com) · fetched via a server-side serverless function (/api/sales-kamsi) using a Shopify Admin access token stored only as a Vercel server-side environment variable — never present in this HTML or any client-side script.
Product ownership source: reports/Kamsi/data/2026-07-14_kamsi-product-allocation.csv — read as-is, not rebuilt from Shopify collections in this dashboard.
Report period: fixed calendar month, Europe/London timezone (DST-aware — Jan/Feb/most of Mar is GMT, late-Mar through Jun is BST; never hardcoded). Select a month tab above; each is a fixed historical period and will never pick up orders from another month.
Channel classification: derived from the customer's first session (customerJourneySummary.firstVisit) — Organic Search, Google Ads/Paid Search, Direct, Social, Email, Referral, or No Journey Data/Unknown when Shopify recorded no session at all.
Kamsi attribution: only line items matching a Kamsi-owned Shopify Product ID/GID (Variant ID/GID preferred when available) contribute to Kamsi's sales figures — mixed-product orders are never fully credited to Kamsi.
Known limitation: the current LEDSone database does not store persistent Shopify Variant IDs for Kamsi's catalog (see allocation CSV), so matching currently operates at Product ID/GID level (Shopify collection membership is per-product, so this does not affect correctness of which orders qualify).
Historical data: these are completed, closed calendar months (Jan–Jun 2026) — the numbers do not change on their own, so there is no auto-refresh or Refresh button. Data is fetched from Shopify once when a month tab is selected and cached server-side (~55 seconds) to avoid re-scanning on repeated page views.
Privacy: no customer email, phone, address, or payment data is fetched, stored, or displayed.
