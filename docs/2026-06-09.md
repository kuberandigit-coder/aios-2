### 5. Blog Builder Bug Fix — Mobile Image Height Not Working

**Bug:** Mobile Height slider in Image blocks had no effect on frontend output.

**Root cause confirmed from code:**
Mobile height styles for single images were pushed into `_inlineStyles` —
a dead array that is defined but NEVER read back and NEVER output to HTML.

The correct array is `_inlineStylesCollector` which IS read and merged into CSS output.
All other working image types (pair, triple, imgtxt) used either `_inlineStylesCollector`
or direct `h +=` injection — both of which work.

**Fix — 2 lines changed in index.html:**

Line 7941 (section block, single image):
  BEFORE: _inlineStyles.push('<style>@media(max-width:768px){#simg-...}</style>')
  AFTER:  h += '<style>@media(max-width:768px){#simg-...}</style>\n'

Line 8176 (imageonly block, single image):
  BEFORE: _inlineStyles.push('<style>@media(max-width:768px){#simg-...}</style>')
  AFTER:  h += '<style>@media(max-width:768px){#simg-...}</style>\n'
